Offline · dependency-light · PDBsum-style interface diagnostics

Every contact in a protein complex, read straight from the coordinates.

Give it any two-chain PDB file — a docked peptide, an antibody-antigen pair, an AMP against its target — and it finds every hydrogen bond, salt bridge, and hydrophobic contact between them, then draws the interface, the secondary structure, and a Ramachandran plot. No Biopython, no PDBsum server required.

01  What it computes

Three contact types, the same three PDBsum draws

Every contact is geometry, not a black box — distances are checked directly against the coordinates in your file.

Hydrogen bonds

Polar atom pairs (N/O/S) positioned close enough to plausibly hydrogen-bond.

2.4–3.5 Å

Salt bridges

Oppositely charged side-chain atoms (Asp/Glu against Lys/Arg/His) within range.

≤ 4.0 Å

Hydrophobic contacts

Non-polar carbon atoms from hydrophobic residues packed against each other.

≤ 4.0 Å

05  Read before you cite

What this is — and isn't

  • Secondary structure comes straight from the PDB file's HELIX/SHEET header records. Raw docking output usually has no header at all, so it will show as coil — that's expected, not a bug.
  • Ramachandran "favored regions" shown here are a simplified, illustrative guide for an at-a-glance check, not a validated statistical potential. For publication-grade validation, cross-check with MolProbity or the structure's wwPDB validation report.
  • Distance thresholds are the same geometry-only heuristics PDBsum/LIGPLOT use — they don't model hydrogen positions or electrostatics, so treat borderline contacts as candidates worth a closer look, not certainties.
